# Fourier Transform Visualizer — Design Document

> 연속 시간 신호의 푸리에 변환을 3차원 복소공간에서 인터랙티브하게 탐험하는 교육용 도구.
> v0.1, 작성일 2026-04-30. 목표 릴리스: 다음 주 월요일까지(약 4일).

---

## 1. Motivation & Goals

### 1.1 왜 만드는가

3Blue1Brown의 "winding machine" 시각화는 푸리에 변환을 직관적으로 보여주는 강력한 도구이지만, 본질적으로 2D 평면 시각화이다. 그래서 다음과 같은 핵심 개념이 그림으로 직접 보이지 않는다:

- **푸리에 변환 결과가 왜 복소수인가** — center of mass의 (x, y) 좌표를 복소수의 (Re, Im)로 해석한다는 추가 설명이 필요함.
- **음의 주파수의 의미** — 회전 방향이 반대인 헬릭스라는 것이 평면에서는 표현이 어렵다.
- **직교성(orthogonality)** — "회전이 멈춘 성분만 적분에서 살아남는다"는 것이 평면 시각화에서는 "원점 부근으로 모인다"라는 간접적 신호로만 드러난다.

이 도구는 **시간축(또는 주파수축)을 한 축, 복소평면(Re, Im)을 다른 두 축**으로 두는 3D 시각화를 메인으로 하여, 위 개념들이 그림에서 직접 읽히도록 한다.

### 1.2 핵심 학습 목표 (Learning Objectives)

도구를 한 시간 가지고 논 학습자가 다음을 *시각적으로* 설명할 수 있어야 한다.

- **L1**: 푸리에 변환은 신호 `x(t)`를 `e^(-jωt)`로 "감은(wound)" 헬릭스를 적분한 것이다.
- **L2**: ω가 신호의 한 성분 `ω_0`와 일치하면, `x(t)·e^(-jωt)`의 그 성분은 회전이 멈추고, 적분에 살아남는다.
- **L3**: ω ≠ ω_0인 다른 성분들은 여전히 회전 중이므로, 충분히 긴 구간에서 적분하면 벡터합이 0에 가까워진다 (직교성).
- **L4**: 따라서 `X(ω)`는 "ω에 대해 정지한 성분의 누적 벡터합"이며, 자연스럽게 복소수 값을 가진다.

### 1.3 비목표 (Non-Goals)

이 도구는 다음을 *하지 않는다*. 명시적으로 빼두는 것이 scope creep을 막는다.

- 이산 시간 신호 처리 (DFT, FFT, 윈도우 함수 등) — 이건 별도의 도구 영역이다.
- 실시간 오디오 입력/스펙트로그램.
- 2D 푸리에 변환 (이미지 처리).
- 라플라스/Z 변환 등 일반화.
- 수치 정확도 벤치마크 — 우리는 교육용이지 신호 처리 라이브러리가 아니다.

## 2. Conceptual Model — 무엇을 그리는가

이 절은 문서의 심장이다. 모든 API는 여기서 정의한 다섯 가지 시각적 요소를 표현하는 수단이다.

### 2.1 다섯 가지 시각적 요소

#### (a) Signal Curve `x(t)`

시간축 위에 그려진 실수값(또는 복소값) 곡선. 사용자가 정의하는 입력. 2D 또는 3D(복소 신호일 경우 Re/Im 분리)로 표시.

#### (b) Winding Helix `e^(-jωt)`

3D 공간에서 시간 축을 따라 복소평면을 회전하는 단위 헬릭스. 각속도는 `-ω`. ω가 클수록 빽빽하게 감긴다. ω가 음수면 반대 방향으로 감긴다.

#### (c) Wound Signal `x(t)·e^(-jωt)`

(a)의 envelope 안에 (b)의 carrier가 들어간 3D 곡선. `x(t)`가 헬릭스의 반지름을 변조한다. 푸리에 적분의 피적분 함수 그 자체.

#### (d) Running Integral Vector — *우리의 핵심 시각화*

**시점 t에서, 원점에서 출발해 (Re, Im) = (∫₀ᵗ Re(x(τ)·e^(-jωτ)) dτ, ∫₀ᵗ Im(x(τ)·e^(-jωτ)) dτ) 까지 향하는 벡터.**

t가 진행됨에 따라 이 벡터의 끝점이 그리는 궤적을 함께 보여준다. 이게 곧 푸리에 적분의 부분합이다.

- 만약 ω가 신호의 한 성분 ω_0와 일치하면: 그 성분은 정지 벡터 → 누적 벡터가 한 방향으로 일관되게 자라남.
- 만약 ω ≠ ω_0이면: 회전 중인 성분 → 한 주기를 돌면 합이 0으로 돌아옴. 누적 벡터의 끝점이 작은 원을 그림.

이 그림이 **"적분이 곧 내적이고, 직교성이 곧 cancel-out"**이라는 메시지를 가장 정직하게 전달한다. 별도의 정규화(center of mass) 트릭 없이.

#### (e) Spectrum `X(ω)` — *v0.2 (모드 B에서 활성화)*

ω를 훑으면서 (d)의 최종값을 점으로 찍어 그린 곡선. v0.1에서는 placeholder로 자리만 남겨둔다.

### 2.2 두 가지 동작 모드

**Mode A — Time-sweep (v0.1 구현 대상)**
- ω 고정. 시간 t를 0에서 T까지 증가시킨다.
- (a)~(d)가 모두 t의 함수로 애니메이션된다.
- 학습 메시지: "이 ω에서 이 신호를 감았을 때, 적분이 어떻게 누적되는가."

**Mode B — Frequency-sweep (v0.2)**
- 적분 구간 [0, T] 고정. ω를 ω_min에서 ω_max까지 증가시킨다.
- 매 ω마다 (d)의 최종 누적 벡터를 한 점으로 찍어 (e) 스펙트럼을 만든다.
- 학습 메시지: "어떤 ω에서 누적 벡터가 크게 살아남는가."

v0.1에서는 모드 A에만 집중하되, API와 UI 레이아웃은 모드 B를 끼워넣을 자리를 남겨두는 것을 원칙으로 한다.

### 2.3 사용자가 직접 만지는 것

모드 A의 풀 인터랙티브 정의:

- **재생/일시정지/스크럽**: 시간 t의 자동 진행 또는 슬라이더 직접 조작
- **ω 슬라이더**: winding 주파수. 신호의 성분 주파수와 일치시켜보는 게 메인 액티비티
- **신호 파라미터**: 표준 신호의 진폭/주파수/위상 등 슬라이더
- **3D 시점 회전**: 마우스 드래그로 카메라 각도 변경 (Plotly 기본 지원)
- **연동된 패널**: 위 다섯 요소가 같은 t·ω 상태를 공유하며 동시에 갱신

## 3. Mathematical Conventions

### 3.1 푸리에 변환 정의

**기본 표기는 Hz(주파수 f) 기반**으로 한다. 학습 직관에 더 맞고("2 cycles/second"), 슬라이더 라벨로도 자연스럽다.

`X(f) = ∫ x(t) · e^(-j 2π f t) dt`

내부 계산은 angular frequency `ω = 2π f`로 통일하되, UI 표기와 사용자 입력은 f를 기본으로 한다. 사용자가 명시적으로 ω 모드로 토글할 수 있다.

### 3.2 적분 근사

연속 시간이라고 했지만 컴퓨터에서 적분은 결국 수치 근사다.

- **유한 구간** [t_min, t_max]에서 균등 샘플링된 N개 점에 대해 단순 사다리꼴 합산.
- 사용자가 t_min, t_max, N을 슬라이더 또는 파라미터로 조절 가능.
- 비감쇠 신호(cos, sin)에 대해 "구간 길이가 늘어나면 ω_0에서의 응답이 점점 뾰족해진다"를 보여줄 수 있는 게 의도된 교육 효과다.

### 3.3 비감쇠 신호의 한계

`cos(2πf₀t)`처럼 ∫|x(t)|dt = ∞인 신호는 엄밀하게는 푸리에 변환이 디랙 델타 함수다. 우리 도구는 그런 분포 이론적 결과를 보여주지 않고, **유한 구간에서의 sinc-like 근사**를 보여준다. 이건 버그가 아니라 의도된 교육적 단순화다. 문서에서 이 점을 사용자에게 명시한다.

## 4. API Layers — 3-Layer Architecture

```
┌──────────────────────────────────────────────────────────┐
│ Layer 3: app/                                             │
│   FourierExplorer (인터랙티브 컴포저)                      │
│   ipywidgets 슬라이더 + 4개 패널 연동                      │
└──────────────────────────────────────────────────────────┘
                       ↑ uses
┌──────────────────────────────────────────────────────────┐
│ Layer 2: viz/                                             │
│   SignalPlot · WindingHelixPlot · WoundSignalPlot ·       │
│   RunningIntegralPlot · SpectrumPlot (placeholder v0.1)   │
│   추상 인터페이스 + Plotly 백엔드 구현                     │
└──────────────────────────────────────────────────────────┘
                       ↑ uses
┌──────────────────────────────────────────────────────────┐
│ Layer 1: core/                                            │
│   Signal · FourierAnalyzer · WoundSignal                  │
│   순수 수학. 렌더러 의존성 0.                              │
└──────────────────────────────────────────────────────────┘
```

각 레이어는 아래 레이어만 알고, 위 레이어는 모른다. 이 분리 덕분에 다음이 가능하다.

- Layer 1만으로도 단위 테스트 가능 (수치 정확도 검증).
- 나중에 Plotly가 아닌 다른 백엔드(Three.js, Manim 등)로 교체 시 Layer 2만 수정.
- Layer 3는 "어떤 슬라이더가 어떤 플롯을 갱신하는가"만 다루면 됨.

### 4.1 Layer 1: core/

순수 수치 계산. numpy 외 의존성 없음.

- **`Signal`**: 시간 도메인 함수의 표현. lambda 또는 표준 팩토리로 생성. `__add__`, `__mul__`, `__rmul__` 지원해 `a*cos + b*gauss` 같은 합성이 자연스럽게 되도록.
- **`signals` 모듈**: `cosine`, `sine`, `complex_exp`, `gaussian`, `rect`, `step`, `decaying_exp` 같은 표준 팩토리.
- **`FourierAnalyzer`**: Signal과 적분 파라미터(t_min, t_max, n_samples)를 받아 분석을 제공. 핵심 메서드는 `wound_at_freq(f)` 또는 `wound_at_omega(ω)`로, 한 winding 주파수에 대해 (c)와 (d)를 한꺼번에 계산해 `WoundSignal` 객체로 반환.
- **`WoundSignal`**: `wound_at_*` 결과를 담는 불변 데이터 객체. `t`, `signal_values`, `wound`(즉 (c)), `running_integral`(즉 (d)), `final_integral`(스칼라) 필드를 제공.

### 4.2 Layer 2: viz/

Plotly 기반. 각 플롯 클래스는 단일 책임:

- **`SignalPlot`**: (a) 시간 도메인 곡선
- **`WindingHelixPlot`**: (b) 단위 헬릭스
- **`WoundSignalPlot`**: (c) 변조된 헬릭스
- **`RunningIntegralPlot`**: (d) 누적 벡터와 그 궤적 — *이 도구의 메인 디시*
- **`SpectrumPlot`**: (e) v0.1에선 placeholder, "Mode B coming soon" 메시지만 표시

각 플롯은 동일한 인터페이스를 따른다: `update(t, omega, signal)` 호출 시 자기 패널을 갱신.

### 4.3 Layer 3: app/

- **`FourierExplorer`**: ipywidgets로 슬라이더 묶음(t, ω, 신호 파라미터)을 만들고, 위 5개 플롯을 같은 상태에 묶음. Jupyter Notebook에서 한 줄로 띄울 수 있도록.

```python
from ftvis import FourierExplorer, signals

sig = signals.cosine(freq=2) + signals.cosine(freq=3)
explorer = FourierExplorer(sig)
explorer.show()  # Jupyter에서 슬라이더 + 4개 플롯이 뜬다
```

## 5. Interaction Model — 사용자 시나리오

### 5.1 시나리오 1: "winding speed가 신호 주파수와 일치할 때"

1. 사용자가 `signals.cosine(freq=2)`를 만든다.
2. `FourierExplorer(sig).show()`를 호출.
3. ω 슬라이더를 0부터 천천히 올린다.
4. f = 2 부근에서 (d) 누적 벡터가 일관된 방향으로 자라는 것을 본다.
5. f를 2에서 살짝 벗어나면, 누적 벡터가 작은 원을 그리며 제자리를 맴도는 것을 본다.

### 5.2 시나리오 2: "신호를 합성하면"

1. `signals.cosine(freq=2) + signals.cosine(freq=3)`로 합성 신호를 만든다.
2. ω 슬라이더를 훑으며 f=2와 f=3 두 곳에서 누적 벡터가 살아남는 것을 본다.
3. 그 사이 주파수에서는 두 성분 모두 회전 중이라 합이 작은 것을 확인.

### 5.3 시나리오 3: "감쇠 신호"

1. `signals.gaussian(sigma=0.5) * signals.cosine(freq=3)`을 만든다.
2. 누적 벡터가 처음에는 자라다가, gaussian envelope가 0에 가까워지면 자라지 않는 것을 본다.
3. ω를 훑으며 f=3 근처에서 매끄러운 가우시안 모양 응답을 확인 (Mode B에서 더 명확히, v0.2).

## 6. Roadmap (4 Days)

각 단계에서 **데모 가능한 산출물**이 있어야 한다는 원칙으로 잘랐다.

### Day 1 — core/ 골격

- `Signal`, `signals` 팩토리 (cosine, gaussian, rect만이라도)
- `FourierAnalyzer.wound_at_freq(f)`가 `WoundSignal`(wound + running_integral 포함)을 반환하도록 구현
- 단위 테스트: `cosine(freq=2)`를 ω=2π·2로 감았을 때 누적 벡터가 직선으로 자라는지 수치 검증

**Done definition**: `python -m pytest`가 통과하고, `cos(2πf₀t)`에 대해 ω = 2πf₀일 때 누적 벡터의 방향이 일정함을 자동으로 확인.

### Day 2 — viz/ 정적 플롯

- Plotly로 (a)(b)(c)(d) 4개 플롯을 각각 그릴 수 있는 클래스
- 정적 스크린샷 4장 — README 후보 이미지
- (e) SpectrumPlot은 placeholder 박스만

**Done definition**: 노트북에서 5줄로 4개 플롯을 띄울 수 있다. 각 플롯이 시각적으로 그럴듯하다.

### Day 3 — app/ 인터랙티브

- ipywidgets 슬라이더 (t, ω, 신호 파라미터)
- `FourierExplorer.show()` 한 번 호출로 모든 게 뜨고 슬라이더가 모든 플롯을 동시에 갱신
- 데모 노트북 `examples/01_intro.ipynb` 작성

**Done definition**: 브라우저에서 슬라이더를 만지면 4개 플롯이 부드럽게 갱신된다. 시나리오 1~3을 사람이 손으로 재현 가능.

### Day 4 — 정리, 문서, 깃허브

- README.md (애니메이션 GIF 1장 포함)
- `requirements.txt`, `setup.py` 또는 `pyproject.toml`
- Open Questions(아래 §8) 중 v0.2로 미루는 것들을 GitHub Issues로 정리
- 깃허브 푸시

**Done definition**: 처음 보는 사람이 README만 읽고 `pip install -e .` → 노트북 실행이 된다.

### 위험 요소와 완화책

- **3D 플롯의 성능**: Plotly의 3D 씬은 점이 많으면 느려진다. 누적 벡터 궤적은 N=500 정도로 시작하고, 필요하면 다운샘플.
- **ipywidgets와 Plotly 3D 연동의 깜빡임**: `FigureWidget`을 써서 부분 업데이트가 가능하도록. 안 되면 `go.Figure` 통째 갱신으로 폴백.
- **하루 일정이 밀릴 경우**: Day 3의 ipywidgets 연동을 단순화 (슬라이더 1~2개로 축소)해서 Day 4의 문서/깃허브 작업을 살린다. "풀 인터랙티브"의 정의는 §2.3에 명시되어 있으나, 우선순위는 ω·t 슬라이더 > 신호 파라미터 슬라이더 > 카메라 회전 순.

## 9. Glossary

- **Signal `x(t)`**: 시간 도메인 입력 함수.
- **Winding**: 신호를 `e^(-jωt)`와 곱해 복소평면을 따라 회전시키는 연산.
- **Wound signal**: `x(t)·e^(-jωt)`. 푸리에 적분의 피적분 함수.
- **Running integral vector**: 시점 t에서의 부분 적분 `∫₀ᵗ x(τ)·e^(-jωτ) dτ`를 복소수로 본 벡터.
- **Mode A / Mode B**: 시간을 진행시키는 모드(ω 고정) / 주파수를 훑는 모드(t 고정).
- **f vs ω**: f는 Hz(cycles/second), ω = 2πf는 rad/second. UI는 f, 내부 계산은 ω.
